Path: %path%

This endpoint allows to update a collection's schema definition. Such schema is validated before any insertion or update of documents and may trigger errors.

If you want more information about these schemas and how to define them, you may visit json-schema.org (if you need some examples).

For example:

{
    "type": "object",
    "properties": {
        "username": { "type": "string" },
        "age": { "type": "number" },
        "data": { "type": "string", "default": "pending..." }
    },
    "required": ["username", "age"]
}

Note: When setting or updating a collection's schema, DocsOnFileDB will validated it before actually setting to avoid inconsistent documents.
